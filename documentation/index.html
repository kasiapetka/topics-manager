Project Specification:

System for handling projects topics. Application is a REST API created with Spring Boot Framework, React Library and with PostgreSQL database. It helps in organizing project groups and sections and managing them. Users appear in three different roles, there is one administrator who can add various teachers. Students can create account with special codes for authentication. Specific requirements and features are shown on multiple diagrams and charts below. Please feel encouraged to get acquainted with the application and its documentation as it may serve You as an instruction for using it.

Requirements:

FUNCTIONAL

Student Module:
  • Log in.
  • Create new account.
  • Manage topics:
    • View students topics.
    • View grades.
    • Add report.
    • View presence.
  • Modify profile and account:
    • Change password.
    • Change email.
    • Change name.
    • Change surname.
    • Delete account.
  • Log out.
Teacher Module.
  • Log in.
  • Manage topics:
    • Add new topic.
    • View and issue grades.
    • View and issue presence.
    • Generate final summary of topic.
  • Manage sections:
    • Add section.
    • Delete section.
    • Modification of section composition.
    • Issue topic to section.
    • Import data about students.
  • Modify profile and account:
    • Change password.
    • Change email.
    • Change name.
    • Change surname.
    • Delete account.
  • Log out.
Administrator Module.
  • Log in.
  • Manage teachers:
    • Add teacher.
    • Delete teacher.
    • Modify teachers account and profile (change password, email, name, surname).
  • Manage students
    • Add students.
    • Delete student.
    • Modify students account and profile (change password, email, name, surname).
  • Modify profile and account:
    • Change password.
    • Change email.
    • Change name.
    • Change surname.
    • Delete account.
  • Log out.
System Module:
    Send email:
  • Notificate student about new grade.
  • Notificate student about being added into new section.
  • Notificate teacher about new report.

NONFUNCTIONAL

Product:
  • One aministrator.
  • System of adding teachers by administrator.
Technology:
  • Database PostgreSQL.
  • REST web application.
  • Frontend with React.
  • Backend with Spring Boot framework.
  • Connection with database.
Security:
  • Passwords securing account access.
  • Users authentication.
  • Encrypting users passwords in database.
Law:
  • Users accept regulations and accept their personal data processing.
  • Displaying and storing data according to RODO.
  • Users accept cookies.
