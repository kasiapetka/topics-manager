Project Specification:

System for handling projects topics. Application is a REST API created with Spring Boot Framework, React Library and with PostgreSQL database. It helps in organizing project groups and sections and managing them. Users appear in three different roles, there is one administrator who can add various teachers. Students can create account with special codes for authentication. Specific requirements and features are shown on multiple diagrams and charts below. Please feel encouraged to get acquainted with the application and its documentation as it may serve You as an instruction for using it.

Workflow

We are trying to use Agile Developement. We are making a lot of meetings where we discuss what is the best for our project. We are trying to follow our Gantt Chart and sometimes we have to make decisions to leave some ideas behind in order to get most important functionality working.

Unfortanently this software isn't made in TDD technique, however we're trying to follow desing patterns and create clean code which is sometimes not that simple. Spring MVC in Topics Manager

This applications is built around Spring MVC Framework, which means that we are explicitly separating Model, View and Controller part of application.

ServiceLayer

Since this application uses Spring MVC framework and its capability of using dependency injection and inversion of control ServiceLayer has been created.

Service Layer specifies interfaces with functions which are neccesary for application to work. Then they these functions are implemented and injected into placere where they are needed. Thanks to Spring's Dependency Incjection it's very easy to implement different functionality and incject it.

Authentication and Authorization

Authentication and authorization is achieved with using two main components: Spring Security and JWT. In Topics Manager every User that is registered into system has login which is email, and password. Aditionally every user has its role: Admin, Teacher or Student.

Spring Security

Spring Security is configured to listen for requests on /api/login endpoint. When it gets correct request it is performing two queries for database. First query checks if given credentials were correct and second is checking which role this user has.

Authorization using Json Web Tokens:

Authorization in Topics Manager Application is implemented using Json Web Tokens. When the user wants to log in and passes correct credentials server responds with a token that holds user email and the time of token expiration. Then whenever the user wants to retrive information from the server client app passes the token he recieved in a HTTP request header. This way of authentication brings the application security on the highest level.

Users

Every user is stored in the same table called "users". Users consist of email, password, id and role which is assigned when user is registered. Every user is assigned to Student or Teacher object except admin who doesn't have name, surname etc.

This structure of tables in database makes it easy for searching and making operations only on accounts or only on teacher or stundent objects. Also this operation was neccesary because there is possibility that Student exists in database without account.
